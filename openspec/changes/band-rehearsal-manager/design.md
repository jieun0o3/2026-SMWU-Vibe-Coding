## Context

신규 Next.js 프로젝트. 밴드 멤버 수 명이 공유해서 쓰는 단일 팀용 웹 앱으로, 복잡한 인증 없이 이름만으로 멤버를 구별한다. MVP이므로 데이터는 서버리스 API Route + JSON 파일(로컬) 방식으로 저장하고, 추후 DB로 교체 가능하도록 인터페이스를 분리한다.

## Goals / Non-Goals

**Goals:**
- 멤버별 불가 날짜 등록 및 조회
- 공연 날짜 + 합주 횟수 입력 → 가능한 날짜 자동 배분
- 합주 일정 캘린더 표시 + 남은 횟수
- 공연 D-day 카운트다운, 당일 합주 시간 알림(배너)
- 합주별 멤버 피드백 + 녹음 파일 업로드(복수)

**Non-Goals:**
- 실시간 동기화 (새로고침으로 최신 데이터 반영)
- 복잡한 인증·권한 관리 (이름 선택으로 멤버 구별)
- 외부 캘린더(Google Calendar 등) 연동
- 녹음 파일 스트리밍 재생

## Decisions

**1. 데이터 저장: API Route + JSON 파일 (fs 기반)**
- 이유: DB 없이 Next.js만으로 동작, 로컬 개발 즉시 가능
- 대안: localStorage → 멤버 간 공유 불가능하여 제외

**2. 합주 자동 배분 알고리즘**
- 입력: 공연 날짜, 합주 횟수, 멤버별 불가 날짜 집합
- 방식: 오늘~공연 전날 사이 전원 가능한 날짜 목록 생성 → 균등 간격으로 N개 선택
- 간격 계산: `Math.floor(가용일수 / (횟수 + 1))` 으로 시작점부터 등간격 배분
- 가용일이 부족하면 경고 반환, 가능한 만큼만 배정

**3. 파일 업로드: multipart/form-data → API Route → /public/recordings/ 저장**
- 파일 크기 제한 20MB per file (Next.js bodyParser 설정)
- 이유: 외부 스토리지 없이 MVP 완성 가능
- 위험: 서버 재시작 시 파일 유지 안 됨 → 추후 S3 교체 여지 남김

**4. 알림: 페이지 배너 방식**
- 브라우저 Notification API는 권한 요청이 필요하여 UX 부담
- 앱 진입 시 오늘 합주 여부 확인 후 상단 배너로 표시

**5. 캘린더: 커스텀 월간 그리드 (외부 라이브러리 없음)**
- 이유: 불필요한 의존성 최소화, Tailwind로 충분히 구현 가능

## Risks / Trade-offs

- [파일 손실] 서버 재배포 시 /public/recordings/ 초기화 → Mitigation: 추후 persistent 스토리지로 교체, MVP에서는 허용
- [동시성] 여러 멤버가 동시에 쓸 때 JSON 파일 충돌 → Mitigation: MVP 규모에서는 무시, 추후 DB 전환
- [알림 신뢰성] 배너는 앱을 열어야만 보임 → Mitigation: MVP 범위 내 허용, 이메일 알림은 Non-Goal
